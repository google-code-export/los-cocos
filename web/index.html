about

cocos2d is a framework for building 2D games, demos, and other graphical/interactive applications. It is built over pyglet. It provides some conventions and classes to help you structure a "scene based application". A cocos application consists of several scenes, and a workflow connecting the different scenes. It provides you with a "director" (a singleton) which handles that workflow between scenes. Each scene is composed of an arbitrary number of layers; layers take care of drawing to the screen (using the pyglet and OpenGL APIs or built it elements like sprites and text), handling events and in general contain all of the game/application logic.

features

  • BSD License Just use it.
  • Pyglet Based No external dependencies.
  • Sprites Fast and easy sprites.
  • Text Rendering
  • Documentation Programming Guide + API Reference + Lots of simple tests showing how to use it.
  • Actions Just tell sprites what you want them to do. Composable actions like move, rotate, scale, etc. Make your own actions too.
  • Effects OpenGL effects like wave and shake
  • Transitions Move from scene to scene with style.
  • Menus Built in classes to create menus

    documentation

    Documentation comes in the form of the programming guide (online HTML, PDF) and API Reference (online HTML, PDF). You can also take a look at the tests in the test folder for simple examples of using the cocos2d functionality.

    mailing lists

    The main forum for cocos2d discussion is the los-cocos-users mailing list. You can browse the mailing list online or provide your email address for immediate or digest updates.

    There are two more mailing lists if you're interested in following the progress of los-cocos development:

    • los-cocos-issues is an announcement list which broadcasts emails from the issue tracker.
    • los-cocos-commits is an announcement list which broadcasts SVN commits.

    issue tracker

    The issue tracker should be used to report any bugs or compatibility issues. You will need a Google account to submit issues.

    wiki

    We have a wiki which is primarily used for discussing future design aspects of los-cocos. Anyone can read this wiki, however you must be a cocos2d developer (have SVN commit access) to write to it.

    subversion repository

    The subversion repository houses the source trunk, documentation, examples and development tools. You can browse the repository or check out a working copy using an SVN client:

    svn checkout http://los-cocos.googlecode.com/svn/trunk/ los-cocos
